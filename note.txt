----------------multiplexing event loop--------------
1. Prepare array of struct pollfd:
	main server listening sockets, currently connected sockets, setting the mask (POLLIN, POLLOUT)

2. Suspension phase:
	poll(), puts the program to sleep until an event occurs

3. Event happens:
	network events (new client or request), changing the revent of the corresponding pollfd.

4. Execution phase:
	loop through pollfds, checks revents bitmask

	If (revents & POLLIN)
	{
		If (listening_set.find(current_fd) != listening_set.end())
			// This is a listening socket -> Call accept()
		Else
			// This is a client socket -> Call recv()
	}

5. Back to 1.


during execution phase, we need to know if its a listening socket or a new connection. to do so

after making sure its a listening socket, to keep track of every message send, even when it connects and cut before the end of the message
we store the different strings using std::map
	std::map<int, std::string> ----> int is the fd of the socket and string is the message.

when accept(), create a new map for the newly connected socket
when poll() flags for POLLIN, add the result of recv() to the string

when recv() returns 0 (EOF) or error, we remove it from the vector of poll fd with the swap and pop, we need to send the string forward
for william to decipher, and then we can remove it from the map.
